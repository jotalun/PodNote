# 后续路线图

版本：`0.6.0`  
更新日期：2026-05-18

## 最近要做

### 1. 线上访问保护

目标：避免公开部署后被别人消耗 DeepSeek 额度。

优先方案：

- Cloudflare Access。
- 简单访问密码。
- `/api/analyze` 请求限流。

### 2. Transcript 获取

目标：减少手动粘贴，支持自动转写。

优先方案：

- 本地 Whisper。
- 第三方语音转文字 API。
- 支持上传音频文件。

### 3. RSS 和播放器增强

目标：让导入后的真实播客更接近日常使用。

需要实现：

- 保存多个订阅源。
- OPML 批量订阅管理。
- 播放失败时提供更明确的错误原因。
- 保存单集播放进度。

### 4. DeepSeek 笔记质量

目标：让 AI 笔记更像真正的知识库条目。

需要优化：

- 提示词按模板拆分。
- 支持「严格摘要」「知识卡片」「研究笔记」「行动清单」。
- 提取人物、书籍、概念、案例。
- 自动生成 Obsidian 双链。

### 5. Obsidian 集成

目标：更自然地写入知识库。

需要实现：

- 选择 Vault 文件夹。
- 自定义导出目录。
- 自定义文件名模板。
- 自动追加 frontmatter。
- 避免覆盖同名文件。

## 中期目标

- OPML 批量导入订阅。
- 本地保存订阅和收听进度。
- DeepSeek API Key 改用系统安全存储。
- 增加任务队列：转写中、分析中、导出完成。
- 支持节目级搜索和 transcript 搜索。

## 长期目标

- 打包成 Mac/Windows 桌面 App。
- 支持小宇宙风格的桌面体验，但底层优先使用开放 RSS。
- 支持多模型：DeepSeek、OpenAI、本地模型。
- 支持批量处理：一次分析多期节目。
- 支持自动同步到 Obsidian、Notion 或本地文件夹。

## 当前不做

- 不直接破解或抓取小宇宙私有数据。
- 不在前端长期保存敏感 API Key。
- 不先做社交功能。
- 不先做账号系统。
