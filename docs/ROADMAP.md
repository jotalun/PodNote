# 后续路线图

版本：`0.17.0`
更新日期：2026-05-18

## 最近要做

### 1. 线上访问保护

目标：避免公开部署后被别人消耗 DeepSeek 额度。

已完成：

- 邀请码登录页。
- Cloudflare Pages 中间件保护页面和 API。
- 用户级转写分钟额度。
- 用户级 DeepSeek 分析次数额度。
- 全站每日转写分钟数上限。
- 全站每日美元预算上限。
- 服务端 transcript 缓存。
- 内测说明页。
- 配置检查页和 `/api/config`。

优先方案：

- Cloudflare Access。
- Cloudflare Turnstile。
- 更完整的后台邀请码管理。

### 2. Transcript 获取

目标：减少手动粘贴，支持自动转写。

已完成：

- 自动读取 RSS 里的公开 transcript。
- 自动发现单集网页里的 transcript / 字幕链接。
- 支持 25MB 以下音频用 OpenAI 生成 transcript。
- 支持超过 25MB 的公开音频用 Deepgram URL 转写。
- 浏览器端 transcript 缓存和每日 300 分钟转写保护。
- Cloudflare KV 服务端 transcript 缓存和额度保护。
- 播放时字幕自动同步高亮。
- 播放器卡片内当前字幕显示。
- 完整字幕时间线移入播放器卡片。
- transcript 下载。
- DeepSeek 默认输出 Obsidian 知识库笔记结构。
- Markdown 下载区只在 DeepSeek 分析成功后显示内容。
- 初始示例节目不再预填 sample transcript。

优先方案：

- D1 会员、订单和账单流水。
- 音频切片转写，作为 Deepgram 不可用时的备用方案。
- 本地 Whisper。
- 支持上传音频文件。

### 3. RSS 和播放器增强

目标：让导入后的真实播客更接近日常使用。

已完成：

- 支持导入小宇宙公开节目页和公开单集页。
- 小宇宙公开音频可以进入播放器和后续转写流程。
- 小宇宙简介里的时间轴会变成章节。

需要实现：

- 针对更多播客网页形态增强 RSS 自动发现。
- 保存多个订阅源。
- OPML 批量订阅管理。
- 播放失败时提供更明确的错误原因。
- 保存单集播放进度。

### 4. DeepSeek 笔记质量

目标：让 AI 笔记更像真正的知识库条目。

需要优化：

- 提示词按模板拆分。
- 支持「严格摘要」「知识卡片」「研究笔记」「行动清单」。
- 提取人物、书籍、概念、案例。
- 自动生成 Obsidian 双链。

### 5. Obsidian 集成

目标：更自然地写入知识库。

需要实现：

- 选择 Vault 文件夹。
- 自定义导出目录。
- 自定义文件名模板。
- 自动追加 frontmatter。
- 避免覆盖同名文件。

## 中期目标

- OPML 批量导入订阅。
- 本地保存订阅和收听进度。
- DeepSeek API Key 改用系统安全存储。
- 增加任务队列：转写中、分析中、导出完成。
- 支持节目级搜索和 transcript 搜索。

## 长期目标

- 打包成 Mac/Windows 桌面 App。
- 支持小宇宙风格的桌面体验，但底层优先使用开放 RSS。
- 支持多模型：DeepSeek、OpenAI、本地模型。
- 支持批量处理：一次分析多期节目。
- 支持自动同步到 Obsidian、Notion 或本地文件夹。

## 当前不做

- 不直接破解或抓取小宇宙私有数据。
- 不在前端长期保存敏感 API Key。
- 不先做社交功能。
- 不先做账号系统。
